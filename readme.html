What it does

This plugin provides two enhancements to the base Zen Cart functionality:

  1. The creation of tag values, similar to the like-named WordPress items, that can be associated with a store's categories and/or products.
  2. Modification to the Zen Cart built-in advanced search processing to allow searches to be performed against these tags.

Zen Tags

The Zen Tags functionality has been heavily modelled after the implementation of the similarly-named feature within a WordPress blog, providing you with another method to group your products.

Database Changes

All database changes are made automatically by an admin-level initialization script, after the installation files are copied to your store's file-system.

Three new tables are provided to manage the Zen Tags. The first (tags) simply associates a unique tag_id with each tag defined for the store. Tags' names are alphanumeric (e.g. A-Z, 0-9) strings. The tags_to_categories and tags_to_products tables identify the mappings between a category or product, respectively, and the store's defined tags.

Field Name Field Description
tag_id The automatically-incrementing, unique ID associated with this tag's name.
language_id Currently hard-coded to 1; provided for (future) multi-language support.
tag_name The textual name of the tag. A tag's name can contain only alphanumeric characters.
Field Name Field Description
tag_mapping_id The index-ID of the element being mapped via this record. Its value depends on the table, either a categories_id or products_id.
tag_id The unique ID that identifies the tags table name being mapped.

The initialization script also creates the Configuration->Category/Product Tags group, allowing you to configure the store-side operation of the Tag Cloud sidebox.

Field Name Field Description
Installed Version Shows the current Zen Tags plugin version.
Enable on Storefront? Identifies whether (true) or not (false, the default, Zen Tags is enabled on your storefront. If set to false, the "Tag Cloud" sidebox will not display and tags will not be part of any advanced-search.
Tag Cloud — Text Size (Smallest) Enter the text (or font) size to be used for the least-used tag. Default: 8.
Tag Cloud — Text Size (Largest) Enter the text (or font) size to be used for the most-used tag. Default: 22.
Tag Cloud — Units Enter the unit type (e.g. pt, px or em) to be used for the font-size overrides. Default: pt.
Tag Cloud — Max Tags Enter the maximum number of tags to be displayed within the Tag Cloud sidebox. Default: 45.
Tag Cloud — Order By Choose how the tags within the Tag Cloud display are to be ordered, either by name or usage count. Default: name.
Tag Cloud — Sort Order Choose how matching tags within the Tag Cloud are sorted, one of ASC or DESC. Default: ASC.
Remove Unused Tags? Choose whether (true) or not (false) tags that are no longer mapped to another entity (product or category) should be removed from the database. Default: true.

Zen Cart Admin Changes

Your Zen Cart admin-console is updated to:

  1. Allow tag entry when editing a category or a product (Catalog->Categories/Products).
    • When you add a tag to a category, that tag is added to all sub-categories and products.
    • When you remove a tag from a category, that tat is removed from all sub-categories and products.
  2. A new (zen_tag_cloud) sidebox is available for positioning within your store (Tools->Layout Boxes Controller).
  3. Configuration for the Zen Cloud are provided in Configuration->Category/Product Tags

Database I/O Manager (DbIo) Integration

ZenTags also provides a DbIo handler to enable easy export and import of your products' tags.

The export creates a 3-column output: v_products_id, v_products_name and v_products_tags — the v_products_tags contains a ^-separated list of current tags for the referenced product.

On import, only the v_products_id and v_products_tags columns are required/used. Leave the column empty and all tags associated with the associated product are removed.

Store-side Changes

This plugin provides the zen_tag_cloud sidebox which, if enabled, displays a collection of tags from your store's configuration. Each tag is displayed as a link to the advanced_search_result page, where all products matching the selected tag are displayed.

ZenTags also enables product-searches to include products with tags that match any search keywords.

Installation

There are core-file overwrites in the plugin, and it makes changes to your database; you should always backup your cart's database and files prior to making any changes. All core-file modifications are marked, search for -lat9.

  1. Copy the files to your cart's file-system, after renaming the YOUR_ADMIN folder to match your Zen Cart's secret admin folder:
    1. /includes/auto_loaders/config.zen_tags.php
    2. /includes/classes/ZenTags.php — used by both admin and storefront processing
    3. /includes/classes/auto.zen_tags_search.php
    4. /includes/extra_datafiles/zen_tags_database_names.php
    5. /includes/languages/english/extra_definitions/zen_tags_extra_definitions.php
    6. /includes/modules/sideboxes/zen_tag_cloud.php
    7. /includes/templates/template_default/sideboxes/tpl_zen_tag_cloud.php
    8. /YOUR_ADMIN/ajax.php — this is a copy of the storefront version, with the AJAX class path modified for use within the admin.
    9. /YOUR_ADMIN/categories.php
    10. /YOUR_ADMIN/product.php
    11. /YOUR_ADMIN/includes/zen_tags.css
    12. /YOUR_ADMIN/includes/auto_loaders/config.zen_tags_admin.php
    13. /YOUR_ADMIN/includes/classes/ajax/index.html
    14. /YOUR_ADMIN/includes/classes/ajax/zcAjaxTagManager.php
    15. /YOUR_ADMIN/includes/classes/dbio/DbIoProductsTagsHandler.php
    16. /YOUR_ADMIN/includes/classes/observers/ZenTagsAdminObserver.php
    17. /YOUR_ADMIN/includes/extra_datafiles/zen_tags_admin_file_database_names.php
    18. /YOUR_ADMIN/includes/init_includes/init_admin_auth.php — modifications to grant the ajax.php module "special" access, so long as an admin is logged in.
    19. /YOUR_ADMIN/includes/init_includes/init_zen_tags.php
    20. /YOUR_ADMIN/includes/languages/english/extra_definitions/zen_tags_extra_definitions.php
    21. /YOUR_ADMIN/includes/languages/english/dbio/DbIoProductsTagsHandler.php
    22. /YOUR_ADMIN/includes/javascript/ajax_tag_list.js
    23. /YOUR_ADMIN/includes/modules/category_product_listing.php — adds the current tag names to the category/product listing.
    24. /YOUR_ADMIN/includes/modules/copy_to_confirm.php — copies any tags associated with the "source" product to its duplicate.
    25. /YOUR_ADMIN/includes/modules/update_product.php — updates the tag values for the product.
    26. /YOUR_ADMIN/includes/modules/product/collect_info.php — gathers the current tag names.
    Note: All core-file changes use the like-named file from the Zen Cart v1.5.5f distribution as the basis, unless otherwise noted.

Un-install

Delete the new files you copied to your store's file-system and restore any backups of the core- and template-file changes, then copy and paste the contents of /zen_tags_uninstall.sql into your admin's Tools->Install SQL Patches to remove the plugin's database changes.
